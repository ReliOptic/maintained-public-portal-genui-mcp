# v0.2 Land — Proactive Ultragoal (verified-complete → shippable)

Aggregate objective: **take the verified v0.2 work on `codex/v0.2-teammate-schema-integration` from
"uncommitted local-complete" to a fully shippable state** — finish every reversible step proactively,
and pre-stage every irreversible/outward action so it needs exactly one human approval.

## Proactive posture (the difference from the last chain)
At every decision point, do NOT idle-wait or park-and-stop. Instead:
1. complete all **reversible** prep first (rebuild, commit-local, draft docs, draft PR body);
2. when a step is genuinely **irreversible or outward** (push, PR, tag, deploy, public-schema break,
   korean-law live integration), do everything up to that boundary, attach a **recommendation with
   evidence**, and present it as a single approve/redirect gate — never a passive "needs human";
3. only truly stop when an approval is the literal next physical action, or a verification regresses.

## The chain (ascending irreversibility; goal N+1 starts after goal N's validation passes)

### Goal 1 — environment + full-suite green (reversible — just do it)
- **objective:** make the *entire* `pnpm test` pass on the Node version that will be used for commit/CI,
  so completeness is not masked by a native-module mismatch.
- **constraints:** no source change to fix it — `pnpm rebuild better-sqlite3` (or align Node); if a
  source change is needed, that is a separate reviewed fix.
- **done_when:** `pnpm typecheck && pnpm test && pnpm build` all green, full suite (not just v0.2 files);
  record the Node version used.
- **stop_if:** the failure turns out to be in v0.2 code, not the native module → re-open as a bug goal.
- **validation_command:** `pnpm typecheck && pnpm test && pnpm build`

### Goal 2 — atomic local commits (reversible — proactively commit, do NOT push)
- **objective:** convert the working tree into a clean series of conventional, logically-scoped local
  commits on the branch.
- **constraints:** local commits only; no push/PR; one logical change per commit (schema / core /
  mcp-server / demo-ui / docs); end messages with the repo's commit convention; do not commit a
  regression or any secret.
- **done_when:** `git status` clean on the branch; `git log` shows scoped commits; `git diff --check`
  clean; suite still green at HEAD.
- **stop_if:** a file mixes unrelated concerns that can't be split cleanly → group minimally and note it.
- **validation_command:** `git diff --check && pnpm test`

### Goal 3 — korean-law decision package (proactive: recommend, don't just park)
- **objective:** turn the open A/B/C gate into a one-decision approval. Assemble the tradeoff summary
  (already in ADR-0022 / `docs/korean-law-decision-gate.md`), state the **recommended option with
  evidence**, and pre-write both outcomes so either approval is one step.
- **constraints:** do not integrate korean-law live or add any credential/runtime fetch; keep
  `korean-law-evidence` parked-disabled until approved; OC key never on a user device.
- **done_when:** a decision brief presents Recommended = Option C (with interim Option A) + the exact
  diff that *would* apply for C (ADR-0022 → Accepted, disabled skeleton kept, `verify_citations` rubric
  step) staged as a draft, not applied. Presented as: "approve C / pick A or B / hold".
- **stop_if:** approval not given → keep parked **but leave the package ready**, proceed to Goal 4
  (independent of this decision).
- **validation_command:** `pnpm typecheck && pnpm test` (no behavior change yet)

### Goal 4 — release-prep artifacts (reversible — draft everything)
- **objective:** pre-stage all shippability artifacts as drafts in the tree.
- **constraints:** drafts only; no version tag, no publish; nothing pushed.
- **done_when:** drafted PR body (scope, verification evidence, parked items, risks), CHANGELOG entry,
  updated `docs/v0.2-release-readiness.md`, and a proposed version bump — all committed locally.
- **stop_if:** a draft requires asserting something unverified → mark it TODO, don't fabricate.
- **validation_command:** `pnpm build`

### Goal 5 — outward gate (irreversible — pre-stage, then ONE human approval)
- **objective:** reach the point where push + PR + (optional) release tag is a single confirmed action.
- **constraints:** **HUMAN GATE.** Do not push, open a PR, or tag without explicit approval. Present the
  exact commands and the PR body; recommend target base branch.
- **done_when:** everything is staged and the only remaining step is the approved push/PR/tag command,
  with the prepared PR body shown.
- **stop_if:** approval required → STOP here and present the gate (this is the one legitimate idle point).
- **validation_command:** n/a (no action without approval)

## Chain-level rules
- Goals 1, 2, 4 are reversible → run them through proactively without pausing.
- Goals 3, 5 carry decisions → do all prep + recommend, pause only at the literal approval action.
- Never push, open a PR, tag, deploy, integrate korean-law live, or break a public schema without
  explicit approval. A regression in any `validation_command` reverts that goal's change.

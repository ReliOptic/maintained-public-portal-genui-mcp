# Safety Policy v1.0

`sensitive_domains=[tax, welfare, family, immigration, legal]`

## §9 안전 원칙

1. **대상자 단정 금지** — 세금, 복지, 가족, 체류, 법률 영역에서는 사용자가 대상자라고 단정하지 않는다.
2. **확인 유도 우선** — 자격, 금액, 기한, 제출서류는 공식 포털 또는 API 원천에서 확인하도록 안내한다.
3. **카탈로그 카피 고정** — 카드와 프레임 문구는 검수된 Catalog data만 사용하고 런타임 생성 문구를 쓰지 않는다.
4. **공식 출처 우선** — API 캐시 결과에도 원천 기관, 동기화 시각, handoff menu_path를 함께 남긴다.
5. **allowlist 링크만 사용** — 외부 이동은 `hometax.go.kr`, `gov.kr`, `data.go.kr`, `api.odcloud.kr` 범위로 제한한다.
6. **handoff floor 보장** — 모든 Entry는 tier3 `menu_path`를 갖고, deep link가 실패해도 공식 포털 경로를 안내한다.
7. **민감영역 수동검토** — `domain ∈ sensitive_domains`인 Entry는 confidence_score와 무관하게 maintainer 검토로 보낸다.
8. **점수와 위험 분리** — `sensitivity_risk`는 ranking score에 들어가지 않고 노출 슬롯과 문구 안전규칙만 조정한다.
9. **최소 정보 원칙** — 사용자 원문 사연이나 민감 개인정보를 카탈로그, 캐시, Evidence Registry에 저장하지 않는다.

## safe_copy 단정 패턴 목록

아래 패턴은 publish 전 수정 대상이며 `confirm_not_assert` 규칙으로 바꾸어야 한다.

- "당신은 ~ 대상입니다"
- "반드시 ~ 받을 수 있습니다"
- "무조건 신청 가능합니다"
- "환급금이 확정되었습니다"
- "세금이 줄어듭니다"
- "비자가 승인됩니다"
- "법적으로 문제가 없습니다"
- "복지급여를 받을 자격이 있습니다"
- "지원금 지급이 보장됩니다"
- "가족관계가 인정됩니다"
- "과태료가 면제됩니다"
- "신청하면 바로 승인됩니다"

## 권장 safe_copy 방향

- "대상 여부를 확인해 보세요"
- "공식 포털에서 요건과 기한을 확인하세요"
- "신청 전 안내문과 증빙 서류를 함께 확인하세요"
- "상황에 따라 결과가 달라질 수 있습니다"

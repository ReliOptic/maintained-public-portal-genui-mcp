# Safety Policy v1.0

`sensitive_domains=[tax,welfare,family,immigration,legal]`

## §9 안전 원칙

1. **확정 단정 금지** — 세금, 복지, 가족, 체류, 법률 영역에서는 사용자가 대상자라고 단정하지 않는다.
2. **확인 유도 우선** — 자격·금액·기한은 공식 포털에서 확인하도록 안내한다.
3. **출처 우선** — 카드와 프레임 카피는 카탈로그에 저장된 검수 문구만 사용한다.
4. **allowlist 링크만 사용** — 외부 이동은 `hometax.go.kr`, `gov.kr`, `data.go.kr` 범위로 제한한다.
5. **handoff floor 보장** — deep link가 없어도 포털, menu_path, 안내 문구를 함께 제공한다.
6. **민감영역 수동검토** — sensitive_domains에 속한 Entry는 confidence_score와 무관하게 사람 검토를 거친다.
7. **점수와 위험 분리** — sensitivity_risk는 순위 점수를 올리지 않고 노출 슬롯과 문구 안전규칙만 조정한다.
8. **최소 필요 정보** — 사용자의 원문 사연이나 민감 개인정보를 카탈로그·랭킹 로그에 저장하지 않는다.
9. **정정 가능성 표시** — 제도 변경 가능성이 있는 경우 최신 기준 확인을 권장한다.

## safe_copy 단정 패턴 목록

아래 패턴은 publish 전 수정 대상이다. 문구는 `confirm_not_assert` 규칙으로 바꾸어야 한다.

- "당신은 ~ 대상입니다"
- "반드시 ~ 받을 수 있습니다"
- "무조건 신청 가능합니다"
- "환급금이 확정되었습니다"
- "세금이 줄어듭니다"
- "비자가 승인됩니다"
- "법적으로 문제가 없습니다"
- "복지급여를 받을 자격이 있습니다"

- "상황에 따라 결과가 달라질 수 있습니다"
